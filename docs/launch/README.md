# Launch checklist

Order of operations that maximizes reach without burning goodwill:

## Day 0 — polish (done)

- [x] npm published (v0.1.0)
- [x] GitHub repo topics, description, homepage set
- [x] README hooks the specific pain point (Cursor Ultra + Kilo/GLM 5.2)
- [x] MIT license, no telemetry, no cloud round-trip

## Day 1 — soft launch

- [ ] Post to **r/cursor** (the highest-signal audience). Wait 30–60 min for
      first replies; answer them.
- [ ] Tweet lead post (see `twitter.md`). Tag @cursor_ai, @kilocode,
      @FireworksAI_HQ.

## Day 2 — Show HN

- [ ] Submit to Hacker News with the draft in `show-hn.md`. Best window in US
      timezones is Tue–Thu, 7-9am Pacific. Don't fake upvotes or ask friends to
      upvote — HN detects it and buries you.
- [ ] Be around for the first 2 hours to answer comments — that's what keeps
      the post on the front page.

## Day 3+ — secondary channels

- [ ] r/LocalLLaMA post (see `reddit.md`) — GLM users specifically care.
- [ ] Drop the link in the **Kilo Code Discord** #show-and-tell channel
      (if that exists) and the Fireworks community.
- [ ] Consider PRing `awesome-cursor`, `awesome-mcp`, and
      `yigitkonur/cli-continues` README's "Community contributions" section to
      mention this as a UX layer.

## Content follow-ups (if any of the above lands)

- Short blog post: "How to actually use Opus 4.8 and GLM 5.2 in one workflow" —
  post to your own blog or dev.to. Use it as the top comment on Reddit / HN.
- Tiny asciinema of `/steal` running (30s max). Embed in README.
